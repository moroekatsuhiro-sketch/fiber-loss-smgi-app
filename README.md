# SM/GI 光ケーブル規格値計算 fix13-history-overwrite-default

## fix13 の変更

- 一度履歴へ保存した入力は、そのまま同じ履歴の上書きモードを維持します。
- 値を変更して再保存しても履歴件数は増えません。
- 「新規入力（現在を退避）」または「上書きモードを終了」を操作した場合のみ、次回保存で新しい履歴を作成します。

## fix12 の目的

現在使用中の fix11-reset-state-verified をベースに、fix11 のリセット対策・軽量バックアップ・ケーブル長直接入力を維持したまま、履歴UIと校正値の自動反映を強化した版です。

## 主な仕様

- ケーブル長 m 直接入力を最優先。未入力時のみ始端/遠端レングスマーク差を使用。
- 測定値・校正値・規格値は小数第3位以下切り捨て、小数第2位表示。
- 入力値全体を自動保存。測定値だけではなく、基本情報・波長別設定・計算状態も保存。
- 起動時に通常ドラフトがあれば自動復元。確認ダイアログは出さない。
- 軽量バックアップは最大2世代、15秒間隔、変更時のみ。
- 入力リセット/新規入力では、直前データをバックアップへ退避してから、作業中状態を全層で明示的に消去。
- 履歴は工事番号ごとの折りたたみ表示。最新の工事番号グループを上に表示。
- 履歴グループ見出しに件数、最新日時、SM/GI、波長、校正値有無を表示。
- 同じ工事番号＋同じケーブル種類＋同じ波長の過去履歴がある場合、空欄の校正値へ自動反映。
- SMの校正値をGIへ流用しない。1310nmを1550nmへ、850nmを1300nmへ流用しない。
- 入力済み校正値は自動上書きしない。
- 始点/終点校正値を後から入力した場合、測定値入力欄と入力結果一覧の校正値表示へ即反映。
- 始端盤名/遠端盤名を後から入力した場合、区間名が手入力でなければ自動反映。
- PWA更新確認は3分ごと。更新適用はユーザー操作時のみ。
