# SM/GI 光ケーブル規格値計算 fix8-auto-update-check-full-draft

## 反映内容

- ケーブル長 m の直接入力欄を追加
- 優先順位を「ケーブル長 m 直接入力」→「始端/遠端レングスマーク差」に変更
- ケーブル長 m が入力されている場合、始端LM・遠端LMが入力されていてもケーブル長 m を最優先で使用
- ケーブル長 m が空欄の場合のみ、始端LM・遠端LMの差からケーブル長を算出
- 計算内部では m を km に変換して規格値計算に使用
- 履歴・控え表示・CSVに、長さ入力方式 / 直接入力ケーブル長 / LM差 / 使用ケーブル長を記録
- 測定値入力欄を `type="text" inputmode="decimal"` に変更し、ブラウザ側の number 入力補正による値消失リスクを低減
- 校正値入力欄も `type="text" inputmode="decimal"` に変更
- 測定値入力・線番メモ入力は即時ドラフト保存に変更
- pagehide / beforeunload / visibilitychange 時にドラフトを強制保存
- 再描画前に測定値を `waveDraft` に退避する処理を強化
- JavaScript小数誤差による切り捨てズレを補正
- OK/NG判定は、測定値も小数第3位以下切り捨て後の値で比較
- 全角数字・全角小数点の数値入力にも一部対応
- PWAキャッシュ名と index.html のバージョンを fix7 に更新

## 計算仕様

規格値 = ケーブル長(km) × ケーブル損失 + 融着点数 × 0.15 + コネクタ数 × 0.35

ケーブル長の優先順位：

1. ケーブル長 m（直接入力・最優先）
2. ケーブル長 m が空欄の場合のみ、始端/遠端レングスマーク差

## マスター

GI 850nm:

- 融着損失 0.15 dB/点
- コネクタ損失 0.35 dB/個

GI 1300nm:
- ケーブル損失 1.00 dB/km
- 融着損失 0.15 dB/点
- コネクタ損失 0.35 dB/個

## 小数処理

- 規格値：小数第3位以下切り捨て、小数第2位表示
- 測定値：小数第3位以下切り捨て、小数第2位表示
- 校正値：小数第3位以下切り捨て、小数第2位表示
- OK/NG判定：切り捨て後の測定値と切り捨て後の規格値で比較
- 負の校正値は0方向へ切り捨て（例：-10.999 → -10.99）

## fix8-auto-update-check-full-draft

- 3分ごとのPWA更新確認を追加。自動再読み込みは行わず、更新が準備できた場合のみ通知バーを表示。
- 起動後・3分ごと・画面復帰時に `registration.update()` を実行。
- 更新確認前、更新適用前、画面離脱前に入力値全体をドラフト保存。
- 保存対象は、工事番号・現場名・区間名・盤名・ケーブル長m・レングスマーク・ケーブル種類・波長・融着点数・コネクタ数・波長別校正値・芯線数・開始線番・測定値・メモ・最新計算状態。
- 更新通知の「更新する」を押した場合のみ、新しいService Workerへ切替。
- アプリ画面と設定画面にバージョン表示を追加。
- 非表示波長の `waveDraft` も復元時に保持するよう修正。
- 保存失敗時はエラー表示し、更新適用を中止。

### 注意

- Android実機でのOS強制終了、サイトデータ削除、ストレージ容量不足、ユーザーによる明示的なリセット/履歴削除は完全防御できません。
- PWA更新確認は「確認」であり、測定中の強制リロードは行いません。
